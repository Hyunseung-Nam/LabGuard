# LabGuard

**연구실 장비 오차 감지 및 모니터링 시스템**

실험 장비의 이상을 실시간으로 감지하여 연구원에게 즉각 알림을 전달합니다.
수일간의 연구 데이터 유실과 고가 시약/샘플 폐기를 방지하는 것이 핵심 목표입니다.

---

## 왜 만들었나

바이오·화학·전자/반도체 분야의 실험은 며칠씩 연속으로 진행됩니다.
심야나 주말에 온도 챔버, 진공 펌프, 계측기 등에서 오차가 발생해도 아무도 모르면,
수일치 데이터가 증발하고 수백만 원어치 샘플이 폐기됩니다.

기존 산업용 솔루션(NI LabVIEW 등)은 구축 비용이 수백~수천만 원에 달해
예산이 부족한 대학 연구실은 여전히 연구원이 밤을 새우며 장비를 지킵니다.

LabGuard는 이 공백을 채웁니다.

---

## 핵심 기능

| 기능 | 설명 |
|------|------|
| 다중 프로토콜 통신 | RS-232, TCP/IP, USB-TMC/GPIB, Modbus 지원 — 구형/신형 장비 모두 연결 |
| DSP 기반 노이즈 필터링 | 이동 평균·EWMA·저역 통과 필터로 순간 스파이크와 실제 오차를 구분, 오탐지 최소화 |
| 실시간 임계치 감지 | 절대값 기준 + 3σ 통계 기준 병용, 디바운싱으로 False Positive 차단 |
| 즉각 알림 | Telegram Bot으로 무음 모드 우회 경고 발송 (그룹 채팅 지원) |
| 시계열 데이터 저장 | TimescaleDB에 모든 측정값 누적 저장, 이력 조회 가능 |
| 웹 대시보드 | 여러 장비 상태를 단일 화면에서 실시간 모니터링 |

---

## 시스템 구조

```
[실험 장비]
    │  RS-232 / TCP/IP / USB / GPIB
    ▼
[Edge Agent]  ──  Raw 데이터 수집 (Python)
    │
    ▼
[DSP Layer]  ──  노이즈 필터링 → 임계치 판정
    │
   / \
  ▼   ▼
[TimescaleDB]   [Telegram 알림]
    │
    ▼
[FastAPI 백엔드]
    │
    ▼
[웹 대시보드]
```

---

## 기술 스택

- **수집:** Python, pyserial, pyvisa, pymodbus
- **신호 처리:** NumPy, SciPy
- **데이터베이스:** TimescaleDB (PostgreSQL)
- **백엔드:** FastAPI, SQLAlchemy (async)
- **알림:** python-telegram-bot
- **배포:** Docker Compose

## 로드맵

**Phase 1 (MVP)** — 단일 장비 알림 시스템
- 타겟 장비 1대 연결, 임계치 초과 시 Telegram 알림 발송

**Phase 2** — 웹 대시보드 + 다중 장비
- 여러 장비 통합 모니터링, 알림 이력 조회, 사용자 인증

**Phase 3** — AI 기반 예지 보전
- LSTM/AutoEncoder 모델로 고장 사전 예측, "48시간 내 고장 확률 85%" 수준의 예측 알림
