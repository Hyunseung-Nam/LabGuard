# LabGuard

**연구실 장비 오차 감지 및 모니터링 시스템**

실험 장비의 이상을 실시간으로 감지하여 연구원에게 즉각 알림을 전달합니다.
수일간의 연구 데이터 유실과 고가 시약/샘플 폐기를 방지하는 것이 핵심 목표입니다.

---

## 왜 만들었나

바이오·화학·전자/반도체 분야의 실험은 며칠씩 연속으로 진행됩니다.
심야나 주말에 온도 챔버, 진공 펌프, 계측기 등에서 오차가 발생해도 아무도 모르면,
수일치 데이터가 증발하고 수백만 원어치 샘플이 폐기됩니다.

기존 산업용 솔루션(NI LabVIEW 등)은 구축 비용이 수백~수천만 원에 달해
예산이 부족한 대학 연구실은 여전히 연구원이 밤을 새우며 장비를 지킵니다.

LabGuard는 이 공백을 채웁니다.

---

## 핵심 기능

| 기능 | 설명 | 상태 |
|------|------|------|
| 다중 프로토콜 통신 | RS-232, TCP/IP, USB-TMC/GPIB, Modbus 지원 — 구형/신형 장비 모두 연결 | 예정 |
| DSP 기반 노이즈 필터링 | 이동 평균·EWMA·저역 통과 필터로 순간 스파이크와 실제 오차를 구분, 오탐지 최소화 | 예정 |
| 실시간 임계치 감지 | 장비별 임계치 설정, 초과 시 AlertEvent 자동 생성 (쿨다운 적용) | ✅ 구현 |
| 즉각 알림 | Telegram Bot으로 이상 감지 즉시 경고 발송 | ✅ 구현 |
| 시계열 데이터 저장 | TimescaleDB에 모든 측정값 누적 저장, 이력 조회 가능 | ✅ 구현 |
| 웹 대시보드 | 장비 등록/삭제, 시계열 그래프, 알림 내역 실시간 모니터링 | ✅ 구현 |
| AI 기반 예지 보전 | LSTM/AutoEncoder 모델로 고장 사전 예측 | 예정 |

## 시스템 구조

```
[실험 장비]
    │  RS-232 / TCP/IP / USB / GPIB
    ▼
[Edge Agent]  ──  Raw 데이터 수집 (Python)
    │
    ▼
[DSP Layer]  ──  노이즈 필터링 → 임계치 판정
    │
   / \
  ▼   ▼
[TimescaleDB]   [Telegram 알림]
    │
    ▼
[FastAPI 백엔드]
    │
    ▼
[웹 대시보드]
```

---

## 기술 스택

- **수집:** Python, pyserial, pyvisa, pymodbus
- **신호 처리:** NumPy, SciPy
- **데이터베이스:** TimescaleDB (PostgreSQL)
- **백엔드:** FastAPI, SQLAlchemy (async)
- **알림:** python-telegram-bot
- **배포:** Docker Compose

---

## 현재 구현 상태

**백엔드**
- FastAPI + SQLAlchemy(async) + TimescaleDB Hypertable
- 장비 등록/조회/삭제 API
- 측정값 저장 + 임계치 초과 시 AlertEvent 자동 생성 (쿨다운 적용)
- Telegram Bot 알림 발송
- Docker Compose 환경 (TimescaleDB + 백엔드)

**프론트엔드**
- 대시보드: 장비 현황 + 최신 측정값, 이상 장비 수, 최근 알림 내역
- 장비 상태 4단계: 정상(초록) / 상한 초과(빨강) / 하한 미달(파랑) / 오프라인(회색) — Telegram 색 체계와 일치
- 이상 알림 후 60초 경과 시 자동 정상 복귀
- 장비 관리: 장비 목록, 등록/삭제, 알림 기반 상태 표시
- 장비 상세: 시계열 그래프 (raw값 + 필터값), 1분/5분/전체 범위 선택, 5초 폴링 갱신
- 알림 내역: 전체 이상 이벤트 로그 (UUID → 장비명 표시)

**시뮬레이터**
- 실제 장비 없이 가짜 측정값 생성 (`backend/simulator.py`)
- 온도계/기압계/전압계 3개 장비, 10% 확률 이상값 주입

**Edge Agent**
- RS-232로 실험 장비에서 측정값을 읽어 LabGuard 서버로 전송 (`edge_agent/`)
- 타겟 장비: JEIO TECH 항온항습기 (WBTC/WEVC 시리즈)
- 드라이버 구조: `arduino_json` (테스트용) / `jeiotech` (실제 장비)
- 포트 자동 감지, 연결 끊김 시 자동 재연결
- Arduino 에뮬레이터 스케치 포함 (`edge_agent/emulator/emulator.ino`)

## 로드맵

**Phase 1 (MVP)** — 단일 장비 알림 시스템
- 타겟 장비 1대 연결, 임계치 초과 시 Telegram 알림 발송

**Phase 2** — 웹 대시보드 + 다중 장비
- 여러 장비 통합 모니터링, 알림 이력 조회, 사용자 인증

**Phase 3** — AI 기반 예지 보전
- LSTM/AutoEncoder 모델로 고장 사전 예측, "48시간 내 고장 확률 85%" 수준의 예측 알림
